# JS → C 语言类比速查表

> 供 Step 3 第四步"关键语法点拨"使用，按需选取相关条目。

---

## 异步编程

| JS 语法 | C 类比 | 点拨说明 |
|:--|:--|:--|
| `async/await` | `xSemaphoreTake(sem, portMAX_DELAY)` | 程序原地挂起，等拿到"令牌"才继续往下跑 |
| `Promise` | 回调函数指针 + 状态机 | 相当于 C 里给任务传一个"完成后回调"的函数指针，只是 JS 把这个模式封装成了对象 |
| `Promise.all([...])` | `xEventGroupWaitBits()` 等待多个事件位同时置位 | 等所有异步任务都完成才继续，任何一个失败就整体失败 |
| `setTimeout(fn, 0)` | `portYIELD()` 主动让出 CPU | 把当前任务放到队列末尾，让其他任务先跑 |
| `setInterval(fn, t)` | `vTaskDelay(t) + while(1)` | 就是一个带固定延时的无限循环任务 |

## 函数与闭包

| JS 语法 | C 类比 | 点拨说明 |
|:--|:--|:--|
| `() => {}` 箭头函数 | 普通函数指针 | 写法更简洁，关键区别是它不改变 `this` 的指向（C 没有 `this`，理解为"自动绑定了外层上下文"） |
| 闭包 (closure) | `static` 局部变量 | 函数执行完后，被捕获的变量不释放，就像 C 函数里的 `static` 变量在调用结束后仍然保留值 |
| 高阶函数 `.map()/.filter()` | `for` 循环 + 条件判断 | `arr.map(fn)` 等价于 for 循环里对每个元素调用 fn，结果存新数组 |

## 对象与结构

| JS 语法 | C 类比 | 点拨说明 |
|:--|:--|:--|
| 解构赋值 `const {a, b} = obj` | `a = obj.a; b = obj.b;` | 语法糖，一行代码完成多个变量赋值 |
| 扩展运算符 `{...obj, x:1}` | `memcpy` + 覆写字段 | 先复制原对象所有字段，再覆盖/追加新字段，原对象不变 |
| 可选链 `obj?.a?.b` | `if(obj && obj->a) obj->a->b` | 避免空指针崩溃，任何一层为 null 就直接返回 undefined 而不报错 |
| 空值合并 `a ?? b` | `a != NULL ? a : b` | 只在 a 是 null/undefined 时才取 b，比 `\|\|` 更精确（`\|\|` 连 0 和空字符串也会取 b） |
| `\|\|` 逻辑默认值 `a \|\| b` | `a ? a : b`（但 a=0 时也走 b） | 当 a 是任何"假值"（0、空串、null、undefined）时取 b |

## 模块与作用域

| JS 语法 | C 类比 | 点拨说明 |
|:--|:--|:--|
| `import/export` | `#include` + `.h` 头文件声明 | `export` 相当于在 .h 里声明函数，`import` 相当于 `#include` |
| `let/const` 块作用域 | 花括号内的局部变量 | 出了花括号就消失，`const` 等同 C 的 `const`，`let` 等同普通局部变量 |
| `var` 函数作用域 | `static` 但范围是整个函数 | 容易造成意外污染，现代 JS 基本不用 |

---

## 常见坑点说明

| 坑 | 类比说法 |
|:--|:--|
| `this` 丢失 | 就像 C++ 的成员函数指针，拿出来单独调用就丢失了对象上下文，箭头函数可以"钉住"this |
| 异步循环里用 `var` | 类似 C 里 for 循环结束后 i 的值已经变了，闭包捕获的是同一个变量 |
| 忘记 `await` | 类似 C 里忘记等待信号量，直接用还没准备好的数据 |
| 忘记 `clearInterval` | 类似 C 里任务创建后忘记删除，内存和 CPU 一直在跑 |
