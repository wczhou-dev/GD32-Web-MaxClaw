## 4. 外设硬件接口区 (0x2001 - 0x6000)

### 4.1 继电器与IO接口
| 地址范围      | 名称            | 说明                                                      |
| :------------ | :-------------- | :-------------------------------------------------------- |
| 0x2001-0x2016 | R1-R22 枚举     | 设备类型枚举 (1=风机组等)                                 |
| 0x2020-0x2029 | DI_1-DI_10 枚举 | DI绑定设备类型                                            |
| 0x2030-0x2033 | AO_0-AO_3 枚举  | AO绑定设备类型                                            |
| 0x4001        | 继电器状态      | uint32 位掩码 (bit0-21，对应R1-R22，**0-based，bit0=R1**) |
| 0x4003-0x4004 | AO1/AO2 反馈    | 风机转速反馈                                              |
| 0x4007        | DI输入状态      | uint16 位掩码 (bit0-9，**0-based，bit0=DI1**)             |
| 0x4008-0x400B | AI1-AI4 反馈    | 电压值 (val/10)                                           |
| 0x5001        | 继电器指令      | uint32 位掩码 (强制启停，**0-based，bit0=R1**)            |
| 0x5003-0x5004 | AO1/AO2 设定    | 风机转速设定                                              |

### 4.2 继电器位序对应关系 (关键：严禁偏移)
> R1 对应 bit0（最低位），R22 对应 bit21。
> relayIndex 传参规则：**0-based**，即 R1=0，R2=1，...，R22=21。
> value=true 表示置位该位（ON），false 表示清零该位（OFF）。
> 写入前必须先读取 0x5001 当前值，合并位图后再写回，严禁覆盖其他位。

### 5.8 告警使能位 (enableBit) — JSON 属性协议

告警使能位**不是 Modbus 寄存器**，而是通过 JSON 属性协议 (`AlarmThresholdSet`) 下发到固件 `App_Save.alarm.enableBit`。屏幕 (HMI) 和测试系统均可通过此协议写入。

**下发格式**:
```json
{
  "AlarmThresholdSet": {
    "lowTemp": 10.0,
    "highTemp": 35.0,
    "lowHumi": 40.0,
    "highHumi": 85.0,
    "CO2": 3000,
    "NH4": 25,
    "lowTempRca": 1,
    "highTempRca": 1,
    "lowHumiRca": 0,
    "highHumiRca": 0
  }
}
```

**enableBit 位映射** (`App_Save.alarm.enableBit`):

| 字段 | 位 | 含义 | 固件默认值 |
| :--- | :--- | :--- | :--- |
| `lowTempRca` | Alarm_Bit_TempLow | 温度低限告警使能 | 1 (开启) |
| `highTempRca` | Alarm_Bit_TempHigh | 温度高限告警使能 | 1 (开启) |
| `lowHumiRca` | Alarm_Bit_HumiLow | 湿度低限告警使能 | 0 (关闭) |
| `highHumiRca` | Alarm_Bit_HumiHigh | 湿度高限告警使能 | 0 (关闭) |
| `CO2Rca` | Alarm_Bit_CO2High | CO2 高限告警使能 | 0 (关闭) |
| `NH4Rca` | Alarm_Bit_NH3High | 氨气高限告警使能 | 0 (关闭) |

> **测试系统写入方式**: 通过 `AteTcpClient.writeConfig()` 发送 JSON 命令，或通过后端 API 调用。

## 6. Modbus RTU 从站地址 (RS485 总线)

### 6.1 温湿度传感器 (16路)

固件使用 **FC03 (读保持寄存器)**，每个从站读 2 个寄存器：

| 寄存器地址 | 内容 | 类型 | 换算 |
|:---|:---|:---|:---|
| 0x0000 | 湿度 (HUM_INDEX) | uint16 | val/10 → %RH |
| 0x0001 | 温度 (TEM_INDEX) | int16 | val/10 → ℃ |

东营版本从站地址表：`[0x01, 0x02, 0x03, 0x07, 0x08, 0x09, 0x50, 0x1A, 0x1B, 0x1C, 0x1D, 0x1E, 0x1F, 0x51, 0x33, 0x34]`

### 6.2 CO2 传感器 (8路)

固件使用 **FC03 (读保持寄存器)**，每个从站读 1 个寄存器：

| 寄存器地址 | 内容 | 类型 | 换算 |
|:---|:---|:---|:---|
| 0x0002 | CO2 浓度 | uint16 | 原值 → ppm |

标准场区从站地址：`[0x05, 0x0A, 0x11, 0x14, 0x15, 0x16, 0x17, 0x52]`

### 6.3 压差传感器 (4路) — FC04 协议

固件使用 **FC04 (读输入寄存器)**，每个从站读 **3 个寄存器**：

| 寄存器偏移 | 内容 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0x0000 (offset 0) | 未使用 | — | 固件忽略此寄存器 |
| 0x0001 (offset 1) | 原始压差值 | int16 (signed) | 使用 0xF000 符号掩码 |
| 0x0002 (offset 2) | 除数指数 | uint16 | 1 = ÷10^1 = ÷10，0 = 原值 |

固件解析公式：`压差(Pa) = diff_press_from_register(tab_reg[1], tab_reg[2])` = `tab_reg[1] / 10^tab_reg[2]`

从站地址：`[0x0B, 0x28, 0x29, 0x2A]`

**模拟器实现注意**：`setSensorValue('press_N', value)` 写入 offset 1 (register 0x0001)，offset 2 固定为 1 (÷10)。offset 0 保留为 0。
