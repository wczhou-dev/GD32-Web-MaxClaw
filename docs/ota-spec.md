# OTA 升级系统开发规格说明书 (V2.0)

## 1. 核心业务流程
本方案采用"中控即服务器 + 设备主动拉取"模式。

1. **上传**：Web端上传固件，Node.js中控重命名为 `SciGeneAI.rbl` 存至 `F:/firmware/`。
2. **配置**：Web端设置目标版本号（int，如201代表2.0.1）通过API告知中控。
3. **触发**：中控写 `0x0101=version`，再写 `0x0100=1`（顺序不可颠倒）。
4. **下载**：环控器检测到 `0x0100==1`，调用内置 `http_ota()` 拉取固件：
```
   http://<BACKEND_IP>:3000/download/SciGeneAI.rbl
```
   > `BACKEND_IP` 从中控 `.env` 文件读取，严禁硬编码。
5. **升级**：下载完成后 Bootloader 执行分区搬运与安装，结束后更新状态寄存器。

## 2. Modbus TCP 寄存器定义
| 地址 (Hex) | 名称         | 类型   | 读写 | 说明                                          |
| :--------- | :----------- | :----- | :--- | :-------------------------------------------- |
| 0x0100     | OTA_TRIGGER  | uint16 | R/W  | 1=触发升级，0=空闲                            |
| 0x0101     | OTA_VERSION  | uint16 | R/W  | 目标版本号（如 201 代表 2.0.1）               |
| 0x0150     | OTA_PROGRESS | uint16 | RO   | 当前进度 0-100%                               |
| 0x0151     | OTA_STATUS   | uint16 | RO   | 0=空闲, 1=下载中, 2=校验中, 3=成功, 0xFF=失败 |

## 3. 中控 API 接口规范

### POST /api/ota/upload
- 功能：固件文件上传
- 格式：`multipart/form-data`
- 存储：重命名为 `SciGeneAI.rbl` 覆盖存至 `F:/firmware/`
- 返回：`{ "success": true, "filename": "SciGeneAI.rbl" }`

### POST /api/ota/start
- Body：`{ "version": 201 }`
- 功能：按序写 `0x0101=201`，再写 `0x0100=1`
- 返回：`{ "success": true, "version": 201 }`

### GET /api/ota/status
- 功能：读取 `0x0150`（进度）和 `0x0151`（状态）
- 返回：`{ "progress": 68, "status": 1 }`

### GET /download/SciGeneAI.rbl
- 功能：为环控器提供固件文件下载
- 说明：静态文件服务，路径固定为 `F:/firmware/SciGeneAI.rbl`

## 4. 环控器 (硬件内置逻辑) 对接说明
- **硬件已内置**：`http_ota()` 函数已集成，无需开发。
- **触发条件**：环控器轮询 `0x0100`，读到1时自动触发下载。
- **进度回传**：下载过程中环控器实时更新 `0x0150`（进度）和 `0x0151`（状态）。

## 5. OTA_STATUS 全量枚举（全链路统一）
| 寄存器值 | 十进制 | 含义     | 前端颜色 |
| :------- | :----- | :------- | :------- |
| 0x00     | 0      | 空闲     | 灰色     |
| 0x01     | 1      | 下载中   | 蓝色     |
| 0x02     | 2      | 校验中   | 橙色     |
| 0x03     | 3      | 升级成功 | 绿色     |
| 0xFF     | 255    | 升级失败 | 红色     |

> 以上枚举为全链路唯一标准，modbus-map.md、ui-spec-all.md、后端代码、前端代码必须全部对齐此表。

## 6. 验收标准
1. 上传后 `F:/firmware/SciGeneAI.rbl` 文件存在。
2. 触发后 Modbus工具查看 `0x0100` 变为1，`0x0101` 为正确版本号。
3. 前端进度条随 `0x0150` 实时平滑更新，校验中阶段显示橙色。
4. 固件CRC校验失败时 `0x0151` 返回 `0xFF`，前端显示红色失败状态。
