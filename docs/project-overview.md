# 项目总览 (project-overview.md)

## 1. 项目背景

本项目是一套基于 Modbus-TCP 的**分布式工业环控系统** Web 测试上位机。
环控器（GD32F407VGT6）直接驱动猪舍风机、水帘、幕帘等动力设备，
中控（Node.js）作为大脑进行全场逻辑调度。

**硬件环境：**
- 中控电脑：Windows 系统，i5-12400，固件存储在 F 盘
- 环控器：GD32F407VGT6，运行 RT-Thread RTOS
- 网络：局域网直连，环控器 IP 段为 10.137.11.xxx
- 通信协议：Modbus-TCP，端口 502

---

## 2. 设计哲学（核心约束，开发前必须理解）

### 绝对稳健 (Robustness)
- 环控器必须具备"网络感知力"。一旦中控心跳丢失，设备必须在 3 秒内
  自动切换至本地 PID 逻辑，**严禁设备因网络故障导致动作锁死**。
- Web 上位机仅作为"观察窗"与"策略配置台"，**核心控制闭环在物理设备本地**。

### 配置与策略解耦
- Web 端主要负责下发目标值（如目标温度 26.5℃），设备端根据目标值
  在本地实时调整 PID 开度。
- **严禁硬编码过多的业务逻辑**，多使用配置化参数（Config-driven）。

### 工业生产需求
- 系统不允许出现单点故障导致整栋舍停机。
- OTA 升级必须支持静默下载、后台校验、自动重启，
  **拒绝任何导致设备变砖的风险**。

---

## 3. 系统架构
```
[环控器 GD32]  <--Modbus-TCP-->  [Node.js 后端]  <--WebSocket-->  [Vue3 前端]
 10.137.11.xxx     局域网           localhost:3000                   浏览器
```

- 后端作为 Modbus Master，主动轮询所有环控器
- 后端同时作为 WebSocket Server，实时推送数据给前端
- 前端只与后端通信，**严禁前端直接处理 Modbus 指令流**
- 固件文件由前端上传至后端，后端提供 HTTP 下载服务给环控器

---

## 4. 需要实现的核心功能

### 功能一：实时环境监控仪表盘（高性能）
- 展示 1#-16# 温湿度、1#-8# CO2、1#-4# 氨气、1#-12# 风速
- 展示舍外温湿度、1#-4# 室内压差
- 数据每 1 秒自动刷新，**系统需支持同时处理十个单元的 Modbus 并发轮询**
- 任何传感器故障、执行器异常，必须在 UI 以"故障码"形式弹出报警
- 设备离线时界面变灰并提示，**严禁忽略任何 Modbus 超时异常**

### 功能二：继电器远程控制（高可观测性）
- 展示 R1-R22 共 22 路继电器的实时状态（ON/OFF）
- 支持通过开关按钮远程控制每一路继电器
- 操作前必须有二次确认弹窗，防止误触
- 任何执行器异常都能在 Web UI 以"故障码"形式弹出报警

### 功能三：OTA 远程升级（安全第一）
- 支持上传 .rbl 固件文件到中控服务器
- 填写目标版本号后一键触发升级
- 实时显示升级进度条和状态（下载中/校验中/成功/失败）
- 必须支持静默下载、后台校验，升级失败必须有明确提示

### 功能四：多设备管理（高扩展性）
- 支持同时连接多台环控器（不同 IP）
- 设备列表从配置文件读取，**未来增加 16# 甚至 32# 舍，
  只需改配置，不需要重构业务逻辑**
- 每台设备独立显示在线/离线状态

## 6. 开发红线（Cursor 严禁违反）

- **严禁**在前端 UI 直接处理复杂的 Modbus 指令流
- **严禁**忽略任何 Modbus 超时异常，必须有明确的 Fallback 提示 UI
- **严禁**硬编码过多业务逻辑，多使用配置化参数
- **严禁**一次性生成所有模块代码，每次只实现一个模块
- **严禁**使用 TypeScript（除非用户明确要求）
- **严禁**在未确认 IP 和端口的情况下硬编码网络参数

---

## 7. 开发优先级

| 优先级 | 模块                 | 说明                     |
| :----- | :------------------- | :----------------------- |
| P0     | shared/constants.js  | 寄存器常量，所有模块依赖 |
| P0     | 后端 Modbus 轮询引擎 | 核心通信，先跑通再说     |
| P0     | WebSocket 数据推送   | 前后端数据桥梁           |
| P1     | 前端环境监控仪表盘   | 最高频使用功能           |
| P1     | 继电器控制表         | 核心控制功能             |
| P2     | 故障报警模块         | 工业级可观测性           |
| P2     | OTA 升级中心         | 重要但不常用             |
| P3     | 多设备管理           | 最后实现                 |
