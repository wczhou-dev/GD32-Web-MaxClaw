# 传感器异常历史数据回退自动化测试方案

> **状态：已合并** — 本文档的关键测试方案已纳入 [传感器测试场景与期望表](../环控器功能/传感器/传感器测试场景与期望表.md)（附录A），此处归档留存。

## 一、测试目标

验证当传感器异常（数据无效）时，系统能否正确使用历史温湿度数据作为替代值，保证控制回路正常运行。测试覆盖 3 组不同时间段的历史数据。

> v3.1 修订重点：本方案不再只判断回退值“非 0 且非 INVALID_VALUE”，而是通过传感器模拟器固定输入 3 组可识别温湿度值，冻结后读取历史缓冲区确认写入，再在异常回退阶段精确断言回退值是否等于对应历史值。启动回退场景必须保证设备重启后传感器仍处于异常状态。

> 执行范围说明：P1 阶段明确采用 **用例 A：启动回退验证** 作为正式自动化测试路径。用例 B（运行时回退）仅保留为源码逻辑说明和后续扩展参考，不纳入本阶段自动化必测范围。

## 二、原理分析

### 2.1 历史数据存储结构

```c
typedef struct {
    float history_InTemp[HISTORY_LOG_MAX];  // 室内温度历史 (25个槽位)
    float history_InHumi[HISTORY_LOG_MAX];  // 室内湿度历史 (25个槽位)
    time_t time_stamp[HISTORY_LOG_MAX];     // 对应时间戳
} sensor_history_t;
```

- 环形缓冲区，25 个槽位，覆盖最近 25 小时
- 每小时触发一次保存（小时变化时）

### 2.2 历史数据保存机制（跨小时冻结）

```c
if (now_time->tm_hour != hour_old) {           // 小时变化触发
    history_InTemp[HISTORY_LOG_MAX - 1] = Sensor_Data.ActualTemp;  // 保存上一小时数据
    time_stamp[HISTORY_LOG_MAX - 1] = time(RT_NULL);               // 时间戳为新小时
}
```

**关键特性：保存的数据与时间戳存在 1 小时偏移**

| 跨小时事件 | 保存的数据来源 | 时间戳 | tm_hour |
|-----------|--------------|--------|---------|
| N:57 → N+1:00 | N 时段传感器值 | N+1:00:xx | **N+1** |

**含义**：tm_hour=N+1 的条目，保存的温度数据实际来自 N 时段。

### 2.3 两层回退机制

| 层级 | 触发条件 | 数据来源 | 执行次数 |
|------|---------|---------|---------|
| **第一层：启动回退** | `flag==0 && ActualTemp==INVALID_VALUE` | `find_current_hour_data()` 按 tm_hour 匹配 | 仅 1 次 |
| **第二层：运行时回退** | `get_HT_CLC_CNT()==0`（无有效采样） | `history_InTemp[0]`（最旧槽位） | 每轮主循环 |

### 2.4 `find_current_hour_data()` 匹配规则

```c
for (i = 0; i < HISTORY_LOG_MAX; i++) {
    localtime_r(&time_stamp[i], &time_data);
    if (time_data.tm_hour == now_time->tm_hour) return i;
}
return -1;
```

- 只匹配 `tm_hour`，不匹配日期
- 因 1 小时偏移：tm_hour=N+1 的条目，温度数据来自 N 时段

## 三、测试方案总览

### 3.1 测试时段选择

选择 3 组不同时段进行测试，确保覆盖不同时间段的历史数据：

| 组号 | 冻结时间（昨天） | 验证时间（今天） | 跨小时 | 模拟温度 | 模拟湿度 | 期望匹配小时 |
|------|----------------|----------------|--------|---------|---------|-------------|
| 第 1 组 | 10:57 | 11:05 | 10→11 | 20.0℃ | 60.0%RH | 11 |
| 第 2 组 | 14:57 | 15:05 | 14→15 | 22.0℃ | 62.0%RH | 15 |
| 第 3 组 | 18:57 | 19:05 | 18→19 | 24.0℃ | 64.0%RH | 19 |

> 说明：3 组模拟值必须故意设置为不同值，不能使用现场自然变化值。这样验证阶段才能精确判断回退到了哪一条历史数据，而不是只判断回退值是否有效。

### 3.2 测试流程

```
=== 冻结阶段 ===
  设置模拟器返回 20.0℃/60.0%RH → 对时到昨天 10:57 → 等待跨小时(10→11) → 读取历史缓冲确认第 1 组冻结完成
  设置模拟器返回 22.0℃/62.0%RH → 对时到昨天 14:57 → 等待跨小时(14→15) → 读取历史缓冲确认第 2 组冻结完成
  设置模拟器返回 24.0℃/64.0%RH → 对时到昨天 18:57 → 等待跨小时(18→19) → 读取历史缓冲确认第 3 组冻结完成

=== 验证阶段（P1 阶段仅执行用例 A）===
  用例 A (启动回退): 每组验证前需重启设备，作为本方案正式验收路径
    设置传感器持续异常 → 重启 → 对时到今天 11:05 → 验证回退值 = 20.0℃
    设置传感器持续异常 → 重启 → 对时到今天 15:05 → 验证回退值 = 22.0℃
    设置传感器持续异常 → 重启 → 对时到今天 19:05 → 验证回退值 = 24.0℃

  用例 B (运行时回退): 仅作为后续扩展参考，本阶段不执行
    模拟异常 → 对时到今天 11:05 → 验证运行时回退来源
    模拟异常 → 对时到今天 15:05 → 验证运行时回退来源
    模拟异常 → 对时到今天 19:05 → 验证运行时回退来源

=== 恢复阶段 ===
  对时回当前真实时间
```

### 3.3 前置条件

| 条件 | 要求 |
|------|------|
| 传感器在线 | 至少 1 个室内温湿度传感器正常工作 |
| 网络连接 | 设备 IP 可达，Modbus TCP 端口 1502 开放 |
| 传感器模拟器 | 能固定返回指定温湿度值，能在重启后保持超时/无效状态 |
| 历史缓冲读取接口 | 能读取最近 3 条 `history_InTemp/history_InHumi/time_stamp`，或通过 MSH/调试寄存器导出 |
| 异常注入接口 | 不能只写只读展示寄存器，必须能让 `get_HT_CLC_CNT()==0` 或启动阶段采样无效 |
| 测试前清理 | 建议提供清空历史缓冲或打印历史缓冲能力，避免旧数据污染 |

## 五、用例 A：启动回退验证（3 组）

### 验证 A1：第 1 组（今天 11:05）

```
1. 设置传感器启动阶段持续异常:
   推荐方式: 传感器模拟器对室内温湿度从站持续不响应
   备选方式: 固件测试钩子设置“启动后采样无效”
   不推荐: 只在重启前写 SENSOR_ACTUAL_TEMP_REG = 0x7FFF
   原因: 如果 ActualTemp 是 RAM 变量，重启后写入值会丢失

2. 重启设备:
   FC 0x06 写 HR18 = 0x55AA
   等待 60 秒, 重新连接

3. 对时到今天 11:05:
   FC 0x10 写 HR10~15 = [今天年, 今天月, 今天日, 11, 5, 0]
   FC 0x10 写 HR16 = 1
   FC 0x03 读 HR16=0, HR17=0 (确认成功)
   FC 0x03 读 HR10~15 (确认设备时间 = 今天 11:05)

4. 验证:
   FC 0x03 读 SENSOR_ACTUAL_TEMP_REG
   预期: ActualTemp ≈ 20.0℃，ActualHumi ≈ 60.0%RH
   (find_current_hour_data() 查找 tm_hour==11, 匹配到第 1 组冻结数据)
```

### 验证 A2：第 2 组（今天 15:05）

```
1. 设置传感器启动阶段持续异常 (同上)
2. 重启设备 (HR18 = 0x55AA)
3. 对时到今天 15:05
4. 验证 ActualTemp ≈ 22.0℃，ActualHumi ≈ 62.0%RH
   (find_current_hour_data() 查找 tm_hour==15, 匹配到第 2 组冻结数据)
```

### 验证 A3：第 3 组（今天 19:05）

```
1. 设置传感器启动阶段持续异常 (同上)
2. 重启设备 (HR18 = 0x55AA)
3. 对时到今天 19:05
4. 验证 ActualTemp ≈ 24.0℃，ActualHumi ≈ 64.0%RH
   (find_current_hour_data() 查找 tm_hour==19, 匹配到第 3 组冻结数据)
```

> 启动回退验收重点：必须证明回退来源是 `find_current_hour_data()` 匹配到的历史条目。只判断 `ActualTemp` 变成有效值不够，因为它可能来自恢复后的实时采样。

| **回退成功** | **ActualTemp/ActualHumi 等于源码规定的历史回退来源值，允许误差按采集精度设置** |

### 8.4 对时跳变规避

当前方案大量使用 Modbus TCP 对时。如果固件历史保存逻辑只判断 `now_hour != hour_old`，则从真实时间跳到昨天 10:57、从 11:00 跳到 14:57、从 15:00 跳到 18:57 时，都可能误触发一次历史保存，污染缓冲区。

建议按优先级采用以下规避方式：

| 优先级 | 方式 | 说明 |
|--------|------|------|
| 推荐 | 增加测试钩子 `history_clear/history_dump/history_freeze_once` | 测试前清空历史，冻结后直接确认写入 |
| 推荐 | 对时后读取历史缓冲，发现非预期条目立即失败 | 防止误保存被忽略 |
| 可选 | 冻结前后保持模拟器值为可识别值 | 即使误保存，也能从数值上定位污染来源 |
| 不推荐 | 只读 HR13 判断跨小时 | 只能证明时间变化，不能证明历史保存正确 |

### 8.5 失败场景

| 失败现象 | 可能原因 |
|---------|---------|
| 历史缓冲全为 0 | 跨小时冻结未正确触发 |
| 回退值为 0 | 历史数据有效性校验未通过 |
| 回退未触发 | ActualTemp 未变为 INVALID_VALUE |
| 回退到非预期值 | 历史缓冲被对时跳变污染，或回退逻辑取错槽位 |
| 启动回退偶发通过 | 重启后传感器恢复实时采样，测试没有保持异常状态 |

## 九、Python 自动化测试脚本

> 说明：以下脚本是自动化开发模板。正式执行前必须补齐 `set_sensor_simulator_normal()`、`set_sensor_simulator_fault()`、`read_history_tail()` 三个接口；如果没有这三个接口，测试只能半自动执行，不能作为可靠的自动化判定依据。

SENSOR_ACTUAL_TEMP_REG = 200  # 示例地址, 需根据实际映射修改
SENSOR_ACTUAL_HUMI_REG = 201  # 示例地址, 需根据实际映射修改
INVALID_VALUE = 0x7FFF

# 3 组测试时段与期望值

    {"name": "第1组", "freeze_hour": 10, "verify_hour": 11, "temp": 20.0, "humi": 60.0},

def modbus_connect():
    client = ModbusTcpClient(DEVICE_IP, port=DEVICE_PORT, timeout=TIMEOUT)
    assert client.connect(), f"无法连接设备 {DEVICE_IP}:{DEVICE_PORT}"
    print(f"[OK] 已连接设备 {DEVICE_IP}:{DEVICE_PORT}")
    return client


def sync_time(client, dt: datetime):
    """对时并确认设备时间"""
    print(f"[SYNC] 对时到 {dt.strftime('%Y-%m-%d %H:%M:%S')} ...")
    result = client.write_registers(HR_TIME_YEAR, [
        dt.year, dt.month, dt.day, dt.hour, dt.minute, dt.second
    ])
    assert not result.isError(), "写入时间寄存器失败"
    result = client.write_register(HR_TIME_TRIGGER, 1)
    assert not result.isError(), "触发对时失败"
    time.sleep(1)
    result = client.read_holding_registers(HR_TIME_RESULT, 1)
    assert not result.isError() and result.registers[0] == 0, f"对时失败"
    result = client.read_holding_registers(HR_TIME_YEAR, 6)
    assert not result.isError(), "读取设备时间失败"
    dev = result.registers
    print(f"[INFO] 设备时间: {dev[0]}-{dev[1]:02d}-{dev[2]:02d} {dev[3]:02d}:{dev[4]:02d}:{dev[5]:02d}")
    assert dev[3] == dt.hour and dev[4] == dt.minute, "对时校验失败!"
    print("[OK] 对时成功")


def wait_cross_hour(client, expected_hour: int):
    """等待跨小时并确认"""
    print(f"[WAIT] 等待跨小时 (预期小时={expected_hour})...")
    for _ in range(200):  # 最多等待 200 秒
        time.sleep(1)
        result = client.read_holding_registers(HR_TIME_HOUR, 1)
        if not result.isError() and result.registers[0] == expected_hour:
            print(f"[OK] 跨小时成功, 当前小时={expected_hour}")
            return
    raise RuntimeError(f"等待跨小时超时, 预期小时={expected_hour}")

def assert_close(actual, expected, name):
    assert actual is not None, f"{name} 读取失败"
    assert abs(float(actual) - float(expected)) <= VALUE_TOLERANCE, (
        f"{name} 不符合预期: actual={actual}, expected={expected}, tolerance={VALUE_TOLERANCE}"
    )


def freeze_group(client, yesterday: datetime, group: dict, group_num: int):
    """冻结一组历史数据"""
    hour = group["freeze_hour"]
    print(f"\n{'='*50}")
    print(f"冻结第 {group_num} 组: 昨天 {hour}:57")
    print(f"{'='*50}")

    set_sensor_simulator_normal(group["temp"], group["humi"])

    freeze_time = yesterday.replace(hour=hour, minute=57, second=0, microsecond=0)
    sync_time(client, freeze_time)

    # 等待跨小时
    expected_hour = hour + 1
    wait_cross_hour(client, expected_hour)

    # 确认设备时间
    result = client.read_holding_registers(HR_TIME_HOUR, 1)
    assert result.registers[0] == expected_hour

    history = read_history_tail(client, count=3)
    # 这里示意按最近一条校验，实际实现可按 tm_hour 搜索。
    latest = history[-1]
    assert latest["tm_hour"] == expected_hour, f"历史小时不匹配: {latest}"
    assert_close(latest["temp"], group["temp"], f"第{group_num}组历史温度")
    assert_close(latest["humi"], group["humi"], f"第{group_num}组历史湿度")

    print(f"[OK] 第 {group_num} 组冻结完成 (tm_hour={expected_hour})")

## 十二、注意事项

1. **跨小时保存的 1 小时偏移**：tm_hour=N+1 的条目，温度数据来自 N 时段
2. **每组等待 3 分钟**：确保传感器完成采样后再跨小时冻结
3. **冻结值必须固定且不同**：建议使用 20.0℃、22.0℃、24.0℃ 这类容易区分的值
4. **不能只判断有效值**：必须断言回退值等于对应历史值，否则可能误判通过
5. **用例 A 每组需重启**：flag 是一次性变量，只有重启才能重置
6. **用例 A 必须保持启动异常**：重启后传感器模拟器仍需超时或返回无效，避免实时采样覆盖历史回退
7. **用例 B 不需重启**：运行时回退每轮主循环都执行，但要确认源码实际取哪个历史槽位
8. **FlashDB 持久化**：历史数据存储在外部 Flash，重启后不丢失
9. **测试后必须恢复时间**：避免影响设备正常运行
10. **对时跳变可能污染历史**：测试前最好清空历史缓冲，冻结后必须读取历史缓冲确认实际写入内容
