# 水帘与幕帘共享免重启热切换优化方案

本方案旨在解决系统在不重启状态下动态关闭水帘共享或幕帘共享时，由于温湿度/开度等缓存残留以及主从判定拦截不严，导致共享逻辑仍然暗中生效的逻辑缺陷。

本方案分为两步执行：
* **第一步：进行水帘的修改**（包括水帘配置应用时的缓存清理、总线数据接收指令拦截、以及主机决策与从机运行的条件过滤）。
* **第二步：进行幕帘的修改**（包括幕帘配置应用时的缓存清理、总线数据接收指令拦截、以及主机最大开度计算的条件过滤）。

另外需要先进行一项全局前置修正：`can_share_adapter_is_*()` 这组主从判断接口不能只看 `type`，还必须同时看对应共享功能是否启用。否则在 HMI 关闭共享后，`type` 仍可能保留主机/从机值，导致业务模块继续按共享主从逻辑运行。

## 全局前置修改：主从判断接口必须同时判断共享使能（修改 `can_share_adapter.c`）

将 `can_share_adapter_is_roller_master()`、`can_share_adapter_is_roller_slave()`、`can_share_adapter_is_water_master()`、`can_share_adapter_is_water_slave()` 的语义统一调整为：

* 幕帘主机：`sRoller.isornot == 1 && sRoller.type == 1`
* 幕帘从机：`sRoller.isornot == 1 && sRoller.type == 2`
* 水帘主机：`sWater.isornot == 1 && sWater.type == 1`
* 水帘从机：`sWater.isornot == 1 && sWater.type == 2`

**代码修改：**
```c
rt_bool_t can_share_adapter_is_roller_master(void)
{
    return ((sRoller.isornot == 1) && (sRoller.type == 1)) ? RT_TRUE : RT_FALSE;
}

rt_bool_t can_share_adapter_is_roller_slave(void)
{
    return ((sRoller.isornot == 1) && (sRoller.type == 2)) ? RT_TRUE : RT_FALSE;
}

rt_bool_t can_share_adapter_is_water_master(void)
{
    return ((sWater.isornot == 1) && (sWater.type == 1)) ? RT_TRUE : RT_FALSE;
}

rt_bool_t can_share_adapter_is_water_slave(void)
{
    return ((sWater.isornot == 1) && (sWater.type == 2)) ? RT_TRUE : RT_FALSE;
}
```

这样关闭水帘共享或幕帘共享后，所有依赖这些 helper 的开机校准、空圈控制、参数加载、窗口位置处理等路径，会自然回退到非共享语义。

## 第一步：进行水帘的修改

### 1. 增加水帘共享关闭时的缓存清理（修改 `can_share_adapter.c`）
在 `can_share_adapter_apply_water_config` 中，当水帘共享使能 `enabled == 0` 时，立即清理运行时相关的缓存，以防旧的从机共享数据残留影响控制计算。

**代码修改：**
```c
rt_err_t can_share_adapter_apply_water_config(rt_uint8_t enabled,
                                              rt_uint8_t type,
                                              rt_uint8_t self_id,
                                              rt_uint8_t node_min,
                                              rt_uint8_t node_max)
{
    rt_err_t result = RT_EOK;

    sWater.isornot = enabled;
    sWater.type = type;
    sWater.selfID = self_id;
    sWater.NodeIdMin = node_min;
    sWater.NodeIdMax = node_max;
    sRoller.type = type;
    sRoller.selfID = self_id;
    sRoller.NodeIdMin = node_min;
    sRoller.NodeIdMax = node_max;

    // 新增：如果关闭水帘共享，立即清理相关缓存数据，防止脏数据干扰决策
    if (enabled == 0) {
        rt_memset(App_Run.Share_Water_Temp, 0, sizeof(App_Run.Share_Water_Temp));
        rt_memset(App_Run.Share_Water_Humi, 0, sizeof(App_Run.Share_Water_Humi));
        rt_memset(App_Run.Share_Water_age, 0, sizeof(App_Run.Share_Water_age));
        App_Run.Share_Slave_Level = 0;
        App_Run.Share_Slave_State = 0;
        App_Run.SlaveNum_Water = 0;
        App_Run.Share_ActualTemp = 0;
        App_Run.Share_ActualHumi = 0;
    }

    if (sWater.isornot == 1) {
        result = can_share_adapter_start();
    }
    can_share_adapter_update_self_id(sWater.selfID);

    return result;
}
```

### 2. 增加水帘总线指令接收处的开启判定（修改 `can_event.c`）
在 `can_rx_share_roller_task()` 的从机分支中，收到主机的 `CMD_GET_WaterLevel` 指令时，增加对本节点水帘共享使能状态 `sWater.isornot == 1` 的校验。只有当自身的水帘共享开启时，才允许应用主机下发的状态和档位。

**代码修改：**
```c
                    case CMD_GET_WaterLevel:
                        if (sWater.isornot == 1) { // 新增：仅在水帘共享开启时应用主机下发的开关及等级
                            App_Run.Share_Slave_State = can_rx_buffer[8];
                            App_Run.Share_Slave_Level = can_rx_buffer[9];
                            // App_Run.Share_Slave_Water_Run = can_rx_buffer[10];

                                "Receive App_Run.Share_State=%d, App_Run.Share_Level=%d, App_Run.Share_Slave_Water_Run=%d",
                                App_Run.Share_Slave_State, App_Run.Share_Slave_Level, App_Run.Share_Slave_Water_Run);
                        }
                        break;
```

### 3. 拦截水帘主从等级决策逻辑（修改 `water_curtain_controller.c`）
修改 `water_curtain_controller_update_level()` 的主从运行环境判定，在开启与关闭共享时提供严密的控制分支隔离：

* **非从机模式（包含未启用共享的普通模式、或主机模式）**：
  判定条件由 `if (can_share_adapter_is_water_slave() == RT_FALSE)` 修改为 `if ((can_share_adapter_is_water_slave() == RT_FALSE) || (can_share_state.water_enabled == RT_FALSE))`。
  即：当关闭水帘共享后，设备即使被配置为从机，也将立刻强制回退到**本地常规温湿度决策水帘等级逻辑**。
* **主机决策段拦截**：
  在主机运行计算段，新增 `&& (can_share_state.water_enabled == RT_TRUE)` 判断。
  即：只有开启共享水帘的情况下，主机才会融合并提升各从机上传的最高等级。
* **从机决策段拦截**：
  在 `else`（从机模式）的分支下，同样加上对共享开启的判定，只有开启共享时才去跟踪主机的档位。

**代码修改：**
```c
void water_curtain_controller_update_level(void)
{
    // ... 前期变量获取保持一致 ...
    if (can_share_adapter_get_state(&can_share_state) != RT_EOK) {
        return;
    }

    // 核心判定修改：如果未启用共享水帘，即便被配置为从机，也应当执行本地独立控制逻辑
    if ((can_share_adapter_is_water_slave() == RT_FALSE) || (can_share_state.water_enabled == RT_FALSE)) {
        wetCurtainLogic.level = 0;
        App_Run.Water_Run     = 0;
        for (int i = 0; i < 5; i++) {
            if ((Sensor_Data.ActualTemp >= wetCurtainLogic.logicMap[i].opTemp) &&
                (Sensor_Data.ActualHumi < wetCurtainLogic.logicMap[i].clHumi) &&
                (pig_age >= wetCurtainLogic.logicMap[i].age)) {
                wetCurtainLogic.level = i;
                App_Run.Water_Run     = 1;
            }
        }

        // 核心判定修改：只有在开启共享水帘的情况下，主机才融合各从机的最高等级
        if (can_share_adapter_is_water_master() && (can_share_state.water_enabled == RT_TRUE)) {
            App_Run.SlaveNum_Water =
                GET_MIN((can_share_state.water_node_max - can_share_state.water_node_min + 1), SHARE_ROLLER_MAX);
            for (int i = 0; i < App_Run.SlaveNum_Water; i++) {
                share_slave_level[i] = 0;
                for (int j = 0; j < 5; j++) {
                    if ((App_Run.Share_Water_Temp[i] >= wetCurtainLogic.logicMap[j].opTemp) &&
                        (App_Run.Share_Water_Humi[i] < wetCurtainLogic.logicMap[j].clHumi) &&
                        (App_Run.Share_Water_age[i] >= wetCurtainLogic.logicMap[j].age)) {
                        share_slave_level[i] = j;
                        App_Run.Water_Run    = 1;
                        Show_age_2[i]        = App_Run.Share_Water_age[i];
                    }
                }
            }

            if (((cnt > 20) || house_empty) && (can_share_state.water_enabled == RT_TRUE)) {
                LOG_I("Self Temp %.1f Humi %.1f Age %d Level %d", Sensor_Data.ActualTemp, Sensor_Data.ActualHumi,
                      pig_age, wetCurtainLogic.level);
                for (int i = 0; i < App_Run.SlaveNum_Water; i++) {
                    LOG_I("Addr %d Temp %.1f Humi %.1f Age %d Level %d", can_share_state.water_node_min + i,
                          App_Run.Share_Water_Temp[i], App_Run.Share_Water_Humi[i], App_Run.Share_Water_age[i],
                          share_slave_level[i]);
                }
            }

            App_Run.Share_ActualTemp = Sensor_Data.ActualTemp;
            App_Run.Share_ActualHumi = Sensor_Data.ActualHumi;
            Show_age_1               = pig_age;
            for (int i = 0; i < App_Run.SlaveNum_Water; i++) {
                if (wetCurtainLogic.level < share_slave_level[i]) {
                    wetCurtainLogic.level    = share_slave_level[i];
                    App_Run.Share_ActualTemp = App_Run.Share_Water_Temp[i];
                    App_Run.Share_ActualHumi = App_Run.Share_Water_Humi[i];
                    Show_age_1               = Show_age_2[i];
                }
            }

            if (((cnt++ > 20) || house_empty) && (can_share_state.water_enabled == RT_TRUE)) {
                cnt = 0;
                LOG_I(
                    "App_Run.Share_ActualTemp=%.1f App_Run.Share_ActualHumi=%.1f App_Run.Share_age=%d "
                    "wetCurtainLogic.level=%d",
                    App_Run.Share_ActualTemp, App_Run.Share_ActualHumi, Show_age_1, wetCurtainLogic.level);
                LOG_I("Share wetCurtainLogic.level=%d App_Run.Water_Run=%d", wetCurtainLogic.level, App_Run.Water_Run);
            }
        }
    } else {
        // 从机模式下，如果共享使能是开启的，才跟踪主机下发的等级
        if (can_share_state.water_enabled == RT_TRUE) {
            if ((App_Run.Share_Slave_Level != 0) && (wetCurtainLogic.level != App_Run.Share_Slave_Level)) {
                wetCurtainLogic.level = App_Run.Share_Slave_Level;
                LOG_I("Share wetCurtainLogic.level=%d", wetCurtainLogic.level);
            }
        }
    }

    if (nWetCurtainLogic != wetCurtainLogic.level) {
        nWetCurtainLogic = wetCurtainLogic.level;
        jet_set_reportbit(PuFl_StateWET);
    }
}
```

### 4. 拦截水帘实际运行主循环（修改 `water_curtain_controller.c`）
`water_curtain_controller_update_level()` 只负责计算水帘档位，真正控制水帘继电器启停的是 `water_curtain_controller_run()`。当前主循环中如果仍按从机路径运行，即使水帘共享已关闭，也可能不执行本地三状态定时启停逻辑。

同时需要修正一个历史耦合点：水帘主循环当前不应使用 `can_share_adapter_is_roller_slave()` 判断幕帘从机状态，而应使用水帘自身的共享状态判断。

**代码修改：**
```c
void water_curtain_controller_run(void)
{
    if (can_share_adapter_is_water_slave() == RT_FALSE) {
        static time_t startTrigger, endTrigger, restartTrigger;

        // 原本地三状态水帘运行逻辑保持不变：
        // 1. 根据月份和时段判断是否允许运行；
        // 2. 根据 App_Run.Water_Run 和 wetCurtainLogic.level 执行开/关/冷却循环；
        // 3. 不满足运行条件时关闭水帘并上报状态变化。
    } else {
        static rt_uint8_t nState = 0;
        if (nState != App_Run.Share_Slave_Level) {
            nState = App_Run.Share_Slave_Level;
            jet_set_reportbit(PuFl_StateWET);
        }
    }
}
```

由于全局前置修改后 `can_share_adapter_is_water_slave()` 已经同时判断 `sWater.isornot == 1`，因此关闭水帘共享后，即便 `sWater.type` 仍保留为从机，也会立即回到本地水帘运行逻辑。

## 第二步：进行幕帘的修改

### 1. 增加幕帘共享关闭时的缓存清理（修改 `can_share_adapter.c`）
在 `can_share_adapter_apply_roller_config` 中，当幕帘共享使能 `enabled == 0` 时，立即清理运行时相关的开度缓存，以防残留的从机开度数据被拿去与主机的本地开度算最大值。

**代码修改：**
```c
rt_err_t can_share_adapter_apply_roller_config(rt_uint8_t enabled,
                                               rt_uint8_t type,
                                               rt_uint8_t self_id,
                                               rt_uint8_t node_min,
                                               rt_uint8_t node_max)
{
    rt_err_t result = RT_EOK;

    sRoller.isornot = enabled;
    sRoller.type = type;
    sRoller.selfID = self_id;
    sRoller.NodeIdMin = node_min;
    sRoller.NodeIdMax = node_max;
    sWater.type = type;
    sWater.selfID = self_id;
    sWater.NodeIdMin = node_min;
    sWater.NodeIdMax = node_max;

    // 新增：如果关闭幕帘共享，立即清理相关开度缓存数据，防止脏数据干扰决策
    if (enabled == 0) {
        rt_memset(App_Run.Share_Position, 0, sizeof(App_Run.Share_Position));
        App_Run.Share_Slave_Position = 0;
        App_Run.Share_ActualPosition = 0;
        App_Run.SlaveNum = 0;
    }

    if (sRoller.isornot == 1) {
        result = can_share_adapter_start();
    }
    can_share_adapter_update_self_id(sRoller.selfID);

    return result;
}
```

### 2. 增加幕帘总线指令接收处的开启判定（修改 `can_event.c`）
在 `can_rx_share_roller_task()` 的从机接收分支中，收到主机的 `CMD_GET_RollerPos` 报文时，增加对本节点幕帘共享使能状态 `sRoller.isornot == 1` 的校验。只有当自身开启了幕帘共享，才允许同步主机的设定位置。

**代码修改：**
```c
                    case CMD_GET_RollerPos:
                        if (sRoller.isornot == 1) { // 新增：仅在幕帘共享开启时应用主机下发的开度
                            App_Run.Share_Slave_Position             = can_rx_buffer[8];
                            App_Save.Window[DO_Roller1].Now_Position = can_rx_buffer[8];
                            store_backup_bkreg(Roller_Position, App_Save.Window[DO_Roller1].Now_Position);
                            jet_set_reportbit(PuFl_Setup_Roller1);
                            LOG_D("Receive App_Save.Window[DO_Roller1].Now_Position=%d",
                                  App_Save.Window[DO_Roller1].Now_Position);
                        }
                        break;
```

### 3. 拦截幕帘主机最大开度融合逻辑（修改 `ventilation_opening_coordinator.c`）
在通风开口协调器的 `ventilation_opening_coordinator_update_shared_inlet_curtain_max()` 函数中，主机（Master）在对所有从机与本机的开度值进行“取最大值”的融合操作前，添加 `&& (can_share_state.roller_enabled == RT_TRUE)` 拦截：

* 只有当幕帘共享开启时，才通过循环各个从机开度更新 `App_Run.Share_ActualPosition` 并覆盖到本机的 Inlet Curtain。
* 若幕帘共享关闭，主机只会独立进行本机的风量/开度同步，忽略从机开度的干扰。

**代码修改：**
```c
void ventilation_opening_coordinator_update_shared_inlet_curtain_max(void)
{
    static rt_uint8_t cnt = 0;
    can_share_adapter_state_t can_share_state;

    ventilation_opening_coordinator_sync_auto_target(OPENING_SERVICE_INLET_CURTAIN,
                                                     &logicEnControl[App_Run.Ventilationlevel].windIn_opa, RT_TRUE,
                                                     ctr_roller, RT_TRUE, "inlet curtain no relay");

    if (can_share_adapter_get_state(&can_share_state) != RT_EOK) {
        return;
    }

    if (((cnt > 20) || (pigsty_info_controller_is_empty())) && (can_share_state.roller_enabled == RT_TRUE)) {
        LOG_I("Self Mode %s Auto_Position %d Set_Position %d",
              (operation_mode_service_is_bit_set(ctr_roller) == RT_FALSE) ? "Auto" : "Manual",
              ventilation_opening_coordinator_get_auto(OPENING_SERVICE_INLET_CURTAIN),
              ventilation_opening_coordinator_get_manual(OPENING_SERVICE_INLET_CURTAIN));
        for (int i = 0; i < App_Run.SlaveNum_Water; i++) {
            LOG_I("Addr %d Position %d", can_share_state.roller_node_min + i, App_Run.Share_Position[i]);
        }
    }

    // 核心判定修改：只有在幕帘共享开启的情况下，主机才融合各从机的开度并计算最大开度
    if (can_share_adapter_is_roller_master() && (can_share_state.roller_enabled == RT_TRUE)) {  // 主控
        App_Run.Share_ActualPosition = 0;
        App_Run.SlaveNum =
            GET_MIN((can_share_state.roller_node_max - can_share_state.roller_node_min + 1), SHARE_ROLLER_MAX);

        for (int i = 0; i < App_Run.SlaveNum; i++) {
            App_Run.Share_ActualPosition = GET_MAX(App_Run.Share_ActualPosition, App_Run.Share_Position[i]);
        }

        if (operation_mode_service_is_bit_set(ctr_roller) == RT_FALSE) {
            App_Run.Share_ActualPosition =
                GET_MAX(App_Run.Share_ActualPosition,
                        ventilation_opening_coordinator_get_auto(OPENING_SERVICE_INLET_CURTAIN));
        } else {
            App_Run.Share_ActualPosition =
                GET_MAX(App_Run.Share_ActualPosition,
                        ventilation_opening_coordinator_get_manual(OPENING_SERVICE_INLET_CURTAIN));
        }

        ventilation_opening_coordinator_set_auto(OPENING_SERVICE_INLET_CURTAIN, App_Run.Share_ActualPosition);
        if (((cnt++ > 20) || (pigsty_info_controller_is_empty())) &&
            (can_share_state.roller_enabled == RT_TRUE)) {
            cnt = 0;
            LOG_I("App_Run.Share_ActualPosition %d", App_Run.Share_ActualPosition);
        }
    }
}
```

### 4. 拦截进风幕帘实际目标位置选择（修改 `inlet_curtain_controller.c`）
`ventilation_opening_coordinator_update_shared_inlet_curtain_max()` 负责计算共享最大开度，但进风幕帘最终执行目标是在 `inlet_curtain_controller_run()` 中选择的。当前逻辑如果只判断 `roller_type == 0`，在关闭幕帘共享但 `roller_type` 仍保留为主机/从机时，仍可能进入共享分支并使用 `App_Run.Share_ActualPosition`。

关闭幕帘共享时，前文会清理 `App_Run.Share_ActualPosition = 0`，如果这里不拦截，存在进风幕帘误取 0 开度的风险。因此目标选择必须同时判断 `roller_enabled`。

**代码修改：**
```c
can_share_adapter_state_t can_share_state;
if ((can_share_adapter_get_state(&can_share_state) != RT_EOK) ||
    (can_share_state.roller_enabled == RT_FALSE) ||
    (can_share_state.roller_type == 0)) {                    // 不共享或共享已关闭
    if (operation_mode_service_is_bit_set(ctr_roller) == RT_TRUE) {  // 使用手动设置的参数
        roller_Position                           = App_Save.Window[DO_Roller1].Set_Position;
        App_Save.Window[DO_Roller1].Auto_Position = App_Save.Window[DO_Roller1].Set_Position;
        if (App_Save.Window[DO_Roller1].Set_Position == 0xFF) {
            roller_Position = App_Save.Window[DO_Roller1].Now_Position;
        }
    } else {  // 自动
        roller_Position = App_Save.Window[DO_Roller1].Auto_Position;
    }
} else {
    roller_Position = App_Run.Share_ActualPosition;
}
```

这样关闭幕帘共享后，进风幕帘会立即回到本机自动/手动目标，不再读取共享最大开度缓存。

## 第三步：补充 CAN 主机接收缓存拦截（修改 `can_event.c`）

前两步已经在配置应用、从机接收主机指令、业务决策端做了拦截。为了避免共享关闭后主机接收线程继续刷新缓存，还需要在 `can_rx_share_roller_task()` 的主机接收分支中，对 `CMD_GET_TEM` 回包按幕帘/水帘共享使能分别过滤。

当前主机线程只在 `sRoller.isornot == 0 && sWater.isornot == 0` 时整体跳过。如果幕帘共享开启、水帘共享关闭，仍可能继续写入 `App_Run.Share_Water_Temp/Humi/age`；反过来也可能在水帘共享开启、幕帘共享关闭时继续写入 `App_Run.Share_Position`。虽然决策端会再次拦截，但缓存仍会被污染，影响状态展示和后续热切换。

**代码修改：**
```c
case CMD_GET_TEM:
    rt_memcpy(&get_tem_idx, &can_rx_buffer[2], 2);

    roller_idx = (get_tem_idx >= sRoller.NodeIdMin) ? (rt_uint8_t)(get_tem_idx - sRoller.NodeIdMin) : 0xFF;
    water_idx  = (get_tem_idx >= sWater.NodeIdMin)  ? (rt_uint8_t)(get_tem_idx - sWater.NodeIdMin)  : 0xFF;

    if ((sRoller.isornot == 1) && (roller_idx < SHARE_ROLLER_MAX)) {
        slave_offline_count[roller_idx] = 0;
        if (can_rx_buffer[8] == 1) {
            App_Run.Share_Position[roller_idx] = can_rx_buffer[9];
            LOG_I("Receive CMD_GET_TEM Ack, Addr %d, App_Run.Share_Position=%d ",
                  get_tem_idx, App_Run.Share_Position[roller_idx]);
        } else {
            App_Run.Share_Position[roller_idx] = 0;
        }
    }

    if ((sWater.isornot == 1) && (water_idx < SHARE_ROLLER_MAX)) {
        if (can_rx_buffer[10] == 1) {
            rt_memcpy(&App_Run.Share_Water_Temp[water_idx], &can_rx_buffer[11], sizeof(float));
            rt_memcpy(&App_Run.Share_Water_Humi[water_idx], &can_rx_buffer[15], sizeof(float));
            rt_memcpy(&App_Run.Share_Water_age[water_idx], &can_rx_buffer[19], sizeof(rt_uint16_t));
            LOG_I(
                "Receive CMD_GET_TEM Ack, Addr %d, App_Run.Share_Water_Temp=%.1f "
                "App_Run.Share_Water_Humi=%.1f App_Run.Share_Water_age=%d",
                get_tem_idx, App_Run.Share_Water_Temp[water_idx],
                App_Run.Share_Water_Humi[water_idx],
                App_Run.Share_Water_age[water_idx]);
        } else {
            App_Run.Share_Water_Temp[water_idx] = 0;
            App_Run.Share_Water_Humi[water_idx] = 0;
            App_Run.Share_Water_age[water_idx]  = 0;
        }
    }

    can_share_adapter_mark_updated();
    break;
```

离线计数 `slave_offline_count` 当前与 `roller_idx` 绑定。如果后续需要支持“只开水帘共享、不开幕帘共享”的主机掉线清缓存，建议进一步把水帘离线计数和幕帘离线计数拆开；本方案先保持现有结构，只阻止关闭项继续写缓存。

## 实施后验证重点

1. 幕帘共享开启、水帘共享关闭：水帘应按本机温湿度/日龄独立计算和运行，`Share_Water_*` 不应继续刷新。
2. 水帘共享开启、幕帘共享关闭：进风幕帘应使用本机自动/手动开度，`Share_Position` 与 `Share_ActualPosition` 不应影响执行。
3. 两者都关闭：CAN 共享线程即使仍存在，也不应影响水帘档位、进风幕帘目标、从机接收主机指令。
4. 从机关闭单项共享后：对应 `CMD_GET_RollerPos` 或 `CMD_GET_WaterLevel` 不应再应用到本地运行状态。
5. 主机关闭单项共享后再重新开启：缓存应从清零状态重新由新回包建立，不能沿用关闭前的旧温湿度或旧开度。
