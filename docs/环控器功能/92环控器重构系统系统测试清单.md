# 92 环控器重构系统测试清单

**固件版本**: 9.0.1
**协议版本**: 1.0.6
**算法版本**: 1.0.6
**目标硬件**: STM32F407 / GD32F407VG
**RTOS**: RT-Thread

## 0. 测试清单评审与执行口径

### 0.1 评审结论

当前清单覆盖面比较完整，已经包含 HMI、参数保存、负压、微正压、开口、传感器、告警、Mesh、CAN、水帘、加热、照明、补偿、目标温度曲线、日龄最小通风等级、间歇通风、轮替通风、季节通风、AO、继电器、喷淋、调试模式、系统管理和综合场景等主功能。

建议继续优化的重点不是单纯增加条数，而是把每个测试项都补齐“前置参数、触发条件、观测信号、合格判据、异常恢复”。环控器系统里很多问题并不会表现为“功能完全不动”，而是表现为状态桶、继电器、AO、HMI 回显、参数文件、Mesh/CAN 上报之间不同步，所以系统测试必须同时看业务结果和内部状态。

### 0.2 测试优先级

| 优先级 | 含义 | 必测范围 |
|---|---|---|
| P0 | 出厂或版本准入必须通过 | 上电、参数保存、传感器、负压/微正压主链路、风机/开口/水帘/加热/告警、继电器/AO 安全输出、断电恢复 |
| P1 | 影响核心养殖控制效果 | 目标温度曲线、日龄关联、间歇/轮替/季节通风、气体/湿度/压力补偿、Mesh/CAN 共享、集中报警器 |
| P2 | 长周期稳定性和边界体验 | 72 小时运行、极限参数、误操作、日志、历史记录、页面切换、调试模式、资源占用 |

### 0.3 通用判定信号

| 判定维度 | 需要观察的内容 | 合格口径 |
|---|---|---|
| HMI 显示 | 页面显示值、状态图标、参数回显、告警提示、历史记录 | 与内部运行态和物理输出一致，刷新无卡死、无明显延迟 |
| 参数保存 | 修改前值、保存返回、重启后值、参数文件、默认值恢复 | 断电重启后仍保持，非法值被拦截或回退，不破坏其他参数 |
| 控制决策 | 通风等级、目标温度、补偿量、运行模式、状态机阶段 | 与配置表、传感器输入和优先级规则一致 |
| 物理输出 | DO 继电器、AO/PWM、开口正反转、风机、水帘泵、加热、照明 | 输出与控制决策一致，互锁正确，无正反转同时动作 |
| 运行态 | `App_Run`、`App_Save`、开口当前位置、风机状态、水帘等级、告警状态 | 状态桶、HMI 回显、实际输出三者一致 |
| 通信 | HMI UART、Mesh、CAN、集中报警器、共享水帘/幕帘 | 通信正常、超时可恢复、离线可报警、恢复后状态同步 |
| 安全恢复 | 停止、复位、断电、通信中断、传感器异常、手自动切换 | 输出进入安全状态，恢复后重新接管，不遗留错误状态 |

#### 0.5.10 综合场景

| 场景 | 组合条件 | 重点判定 |
|---|---|---|
| 夏季高温高湿 | 高温、高湿、CO2 超标 | 通风优先级、水帘湿度抑制、气体补偿是否协调 |
| 冬季低温 | 低温、最小通风、加热开启 | 加热与最小通风并存，避免过通风 |
| 传感器异常 | 主温度离线、压力离线、气体离线 | 告警、保护输出、备用值或禁用策略正确 |
| 通信异常 | HMI 断连、Mesh 断连、CAN 断连 | 本地控制不中断，通信恢复后状态同步 |
| 断电恢复 | 风机/开口动作中断电 | 上电后参数、开口位置、输出状态安全恢复 |
| 远程接管冲突 | 中控接管时本地手动操作 | 优先级清晰，超时释放后本地恢复 |
