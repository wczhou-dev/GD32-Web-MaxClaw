# GD32 环控系统多环境部署与网络配置手册

本手册用于指导在不同办公环境或网络网段下，如何快速配置并运行前后端系统。

---

## 1. 快速启动指南

### 后端 (Backend)
- **目录**: `backend/`
- **启动命令**: `npm run dev`
- **验证**: 终端出现 `[Connect] Connecting to devices...` 表示开始尝试连接硬件。

### 前端 (Frontend)
- **目录**: `frontend/`
- **启动命令**: `npm run dev`
- **访问**: 默认地址为 `http://localhost:5173`。

### 硬件端 (Embedded)
- **方式**: 烧录完固件后，通过网线连接至交换机。
- **验证**: 串口日志出现 `Heartbeat: X` 表示已成功与后端握手。

---

## 2. 跨环境部署：关键 IP 修改清单

当您从“办公室 A”移动到“办公室 B”，导致电脑和板子的 IP 网段（例如从 `192.168.10.x` 变为 `192.168.1.x`）发生变化时，请按顺序修改以下位置：

### 第一步：修改硬件端 IP (rtconfig.h)
在单片机工程中，修改静态 IP 为当前局域网下的空闲地址。
- **文件**: `rtconfig.h`
- **关键宏**:
  ```c
  #define RT_LWIP_IPADDR "192.168.10.233"  // 修改为新网段IP
  #define RT_LWIP_GWADDR "192.168.10.1"    // 修改为对应网关
  ```
- **临时修改（无需重烧）**: 在串口 MSH 终端输入：
  `ifconfig e0 192.168.10.233 192.168.10.1 255.255.255.0`

### 第二步：修改后端配置 (.env)
后端需要知道去哪里找板子，以及自己监听哪个物理网卡。
- **文件**: `backend/.env`
- **关键项**:
  ```env
  DEVICE_IP=192.168.10.233   # 必须与第一步中板子的 IP 一致
  LOCAL_IP=192.168.10.127    # 修改为您电脑当前获取到的以太网 IP
  ```

### 第三步：前端自动适配
前端代码已配置为 `window.location.hostname` 自动识别模式：
- **原理**: 您在浏览器输入什么 IP 访问前端，它就会自动去连该 IP 下的 3001 端口请求数据。
- **注意**: 如果您在 `192.168.10.127` 的电脑上运行，请确保另一台查看数据的设备也访问这个 IP。

## 3. 防坑检查清单 (Checklist)

1. **防火墙**: 换环境后，Windows 可能会重新提醒“是否允许 Node.js 访问网络”，请务必点击**允许**，或者直接关闭防火墙进行调试。
2. **IP 冲突**: 如果 Ping 不通，请立即拔掉板子网线再 Ping 一次。如果依然能 Ping 通，说明该 IP 被占用，请换一个 IP（如 `.234`, `.235`）。
3. **网段一致性**: 确保电脑的 IP（可通过 `ipconfig` 查看）和板子的 IP 前三段（如 `192.168.10.x`）完全一致。
4. **VMware 冲突**: 如果电脑装有虚拟机，建议禁用 VMware 的虚拟网卡（VMnet1/VMnet8），防止后端程序把数据发到虚拟网络里去。
